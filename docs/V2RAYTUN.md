# v2rayTun + бот «Скинь мне»

## Почему VPN «включён», а бот не работает

**v2rayTun** в режиме **TUN/VPN** пускает через туннель в основном трафик системы/браузера.  
**Node.js (бот)** по умолчанию **не использует** этот туннель — ему нужен **локальный HTTP/SOCKS прокси** на `127.0.0.1`.

Порт `7890` (Clash) у v2rayTun обычно **не открыт** — отсюда ошибка.

---

## Что сделать в v2rayTun

1. Откройте **Настройки** (Settings).
2. Найдите и **включите** одно из:
   - **HTTP Proxy** / **HTTP прокси**
   - **Additional HTTP traffic proxying**
   - **System Proxy** / **Системный прокси**
3. Запомните **порт** (часто `10808`, `10809`, `1080`).
4. В `.env` укажите:

```env
# Порт из v2rayTun → «Порт прокси» (например 10801). Обязательно http://
TELEGRAM_PROXY=http://127.0.0.1:10801
```

**Не пишите** `HTTPS://` — только `http://127.0.0.1:ПОРТ`.

5. Проверка:

```cmd
node scripts\test-telegram-network.mjs
```

Должно быть: `OK proxy ...`

6. Запуск бота:

```cmd
scripts\start-bot.cmd
```

Строка: `Прокси: ...` и `Bot is running @...`

---

## Автопоиск

Бот при старте сам проверяет:

- `TELEGRAM_PROXY` из `.env`
- системный прокси Windows (если v2rayTun его включил)
- типичные порты v2ray: `10808`, `10809`, `1080`, `20171`

Если ни один не работает — включите HTTP-прокси в v2rayTun или запустите бота на VPS.
